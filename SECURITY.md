# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

Если вы обнаружили уязвимость в системе, пожалуйста, сообщите нам об этом ответственно.

### Как сообщить

1. **НЕ создавайте публичный issue** для уязвимостей безопасности
2. Отправьте email на: depfin@yakutsk.org
3. Укажите в теме письма: "SECURITY: [краткое описание]"

### Что включить в отчет

- Описание уязвимости
- Шаги для воспроизведения
- Потенциальное влияние
- Предложения по исправлению (если есть)

### Что ожидать

- **Подтверждение получения**: в течение 48 часов
- **Первоначальная оценка**: в течение 7 дней
- **Исправление**: в зависимости от серьезности (1-30 дней)
- **Публичное раскрытие**: после выпуска исправления

## Безопасность инфраструктуры

### Рекомендации по развертыванию

1. **Используйте HTTPS** для production
2. **Настройте firewall** для ограничения доступа
3. **Регулярно обновляйте** зависимости
4. **Используйте сильные пароли** для БД и Grafana
5. **Ограничьте доступ** к Prometheus и Grafana
6. **Включите rate limiting** на API endpoints
7. **Настройте backup** базы данных

### Переменные окружения

Никогда не коммитьте:
- Пароли баз данных
- API ключи
- Секретные токены
- Приватные ключи

Используйте `.env` файлы и добавьте их в `.gitignore`.

### Docker Security

```bash
# Запускайте контейнеры с ограниченными правами
docker-compose up -d --no-build --force-recreate

# Регулярно обновляйте образы
docker-compose pull
docker-compose up -d
```

### Database Security

```sql
-- Создайте отдельного пользователя для приложения
CREATE USER finance_app WITH PASSWORD 'strong_password';
GRANT SELECT, INSERT, UPDATE ON ALL TABLES IN SCHEMA public TO finance_app;

-- Не используйте postgres суперпользователя в production
```

## Известные ограничения

- Система не предназначена для обработки PII без дополнительной защиты
- Rate limiting должен быть настроен на уровне reverse proxy
- CORS настроен для development, требует настройки для production

## Обновления безопасности

Мы регулярно проверяем зависимости на уязвимости:

```bash
# Backend
cd backend
go list -m all | nancy sleuth

# Frontend
cd frontend
npm audit
```
